# 학사일정 관리 서비스 – 리팩토링 프로젝트

## 프로젝트 소개

기존 학사일정 서비스는 단일 구조 중심으로 개발되어 유지보수성과 협업 효율이 낮았습니다. 기능은 동작했지만, 레이어 분리가 되어 있지 않아 수정 시 전체 코드 흐름을 매번 확인해야 했고 팀 단위 개발에서는 충돌과 혼선이 자주 발생했습니다.

이 프로젝트는 이러한 협업에서의 불편함을 개선하기 위해 진행된 리팩토링 프로젝트입니다. 기존 레포지토리를 분석한 뒤, **구조 개선 · 공통 에러 체계 구축 · 문서화 · DB 전환 · 자동 배포**까지, 실무 기준의 협업 환경을 목표로 재구성했습니다.

- 개발 기간: 2023.11.13 ~ 2023.11.25
- 리팩토링 기간: 2025.11.27 ~ 2025.12.2

<br>

## 기술 스택

- **Backend**: Spring Boot, Java 21
- **Database**: MySQL, H2(테스트용)
- **Infra**: AWS EC2
- **Build/Deploy**: GitHub Actions, Gradle

<br>

## 주요 기능

- 학사 일정의 카테고리 기반 분류 및 조회 기능 (해당 기능을 기반으로 전체 구조 리팩토링 진행)

<br>

## 리팩토링 목표

- 유지보수성과 확장성을 고려한 계층형 구조로 전환
- 실 서비스 환경을 고려한 MySQL 기반 구조 확립
- API 명세 자동화를 통해 팀 개발 환경에서도 활용 가능한 문서 기반 마련
- 에러 처리 일원화 및 공통 응답 포맷 도입
- 리팩토링 과정에서 테스트 기반 안정성을 확보
- CI/CD 구축으로 일관된 배포 환경 마련

<br>

## 주요 개선 사항

### 1) 아키텍처 재구성

- Controller / Service 레이어 분리
- 도메인 모델과 DTO를 명확히 분리

### 2) 에러 코드 체계 구축

* GlobalExceptionHandler 도입
* ErrorCode Enum 설계로 에러 케이스 명확화
* API 공통 응답 구조 정립

### 3) DB 전환 (H2 → MySQL)

- 운영 환경 기준으로 DB 구조 재설계
- Entity 구조 정리

### 4) Swagger 적용

- API 문서 자동 생성
- 협업 상황에서도 명세 공유가 용이한 환경 마련

### 5) 테스트 코드 도입

- Controller · Service 레이어 테스트 작성
- 주요 비즈니스 로직의 정상/예외 흐름 검증
- 입력값 검증 / Request 파라미터 유효성 테스트
- 리팩토링 이후 변경된 구조에서 기능 일관성과 안정성 확인

### 6) CI/CD 파이프라인 구축

- GitHub Actions 기반 자동 빌드·배포
- 환경 분리(dev/prod) 적용

<br>

## Before & After 비교

| 항목    | Before      | After                      |
|-------|-------------|----------------------------|
| 구조    | 단일 파일       | 계층형 아키텍처                   |
| DB    | H2          | MySQL                      |
| 문서화   | 엑셀 기반 수동 전달 | Swagger 자동 문서              |
| 에러 처리 | 문자열 위주      | ErrorCode + Global Handler |
| 테스트   | 없음          | 테스트 적용 후 동작 일관성 보장         |
| 배포    | 수동 배포       | 자동 배포(CI/CD)               |

<br>

## 회고

이번 리팩토링을 통해 단일 구조로 운영되던 기존 코드가 협업과 유지보수에 얼마나 취약한지 직접 체감할 수 있었다.

계층 분리, 공통 에러 처리, 문서화, 테스트, 자동 배포를 적용하면서 구조적인 품질이 눈에 띄게 개선되었고, 변경에도 흔들리지 않는 형태로 안정성을 확보할 수 있었다.

특히 문서화·테스트·배포 자동화는 협업 환경에서 필수적인 요소임을 다시 한 번 느낄 수 있었다. 앞으로는 트랜잭션 경계 설정과 테스트 범위를 더 넓혀 완성도를 점진적으로 높이고자 한다.
